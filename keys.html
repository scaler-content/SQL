Primary Key — Every row must have a unique identifier. In Users, user_id is the PK. No two users share the same user_id. It can never be blank. It's like a fingerprint for each row.
Foreign Key — The user_id in the Rides table points back to the Users table. It doesn't create a new ID — it reuses the one that already exists. This is how two tables stay connected.
Unique Key — No two users can register with the same email. Unlike Primary Key, a Unique Key allows one NULL value. It's a rule the database enforces automatically to prevent duplicate accounts.
