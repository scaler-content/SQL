Key concepts to remember
Primary Key

The unique identity

Every row gets one PK. It never repeats, never changes — like your Aadhaar number. It belongs to one person, forever.

customer_id: C001 → always Priya · only Priya
Foreign Key

The bridge between tables

A FK in one table points to a PK in another. This is how tables talk to each other — and the reason a JOIN is possible.

orders.customer_id = customers.customer_id
Nullable

Empty is valid — sometimes

NULL means no value was stored. A customer who hasn't rated yet has NULL in the rating column. Always handle NULLs carefully — they break averages silently.

AVG(rating) skips NULLs · COUNT(*) includes them
